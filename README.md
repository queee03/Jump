# 功能点

- 基本需求

1. 整体场景的生成设置
2. 物体的降落逻辑
3. bottle 的跳跃过程逻辑（物理引擎）
4. bottle 的跳跃结果逻辑（碰撞检测）
5. 实现分数+1 累计
6. 分数上传 CloudStorage

- 动画优化版本

1. 降落的弹性
2. 粒子聚集的效果
3. 跳跃尾部动态优化
4. 跳跃失败动画
5. 物体降落弹性动画

- 社交优化版本

1. 分享到微信群
2. 分享给朋友
3. 增加排行榜
4. 观战模式

- 完善版本

1. 增加开始页面
2. 优化积分策略
3. 增加音乐效果
4. 增加 block 类型和纹理类型

# 项目结构

- ES6 开发
- 游戏逻辑使用面向对象进行模块化开发（相对于数据驱动图像）
- 游戏整体用 MVC 进行控制和管理
